# dev/docs 使用指南

本目录保存项目的正式过程文档，用于让 Codex 和项目成员恢复上下文、追踪决策、检查问题和验证结果。

## 文档索引

| 文档 | 用途 | 更新时机 |
|------|------|---------|
| `项目管理文档.md` | 项目目标、职责、流程、里程碑 | 流程或计划变化时 |
| `需求变更日志.md` | 需求、澄清、关键决策的单一事实来源 | 每次关键决策后 |
| `问题跟踪.md` | bug、风险、调查项和修复状态 | 发现/修复/验证问题时 |
| `测试记录.md` | 自动/手动测试记录 | 每次测试后 |
| `里程碑检查清单.md` | 阶段验收标准 | 开始或完成里程碑时 |
| `系统结构.md` | 架构、目录、页面、API、数据关系 | 结构变化时 |
| `标签字段字典.md` | 字段定义、标签命名空间、受控词表 | 标签体系、字段设计或分类规则变化时 |
| `数据库设计.md` | RSS 入库、文章正文、AI 打标、证据和候选标签的 SQLite schema | 数据入库、打标流程或表结构变化时 |

## 新会话恢复上下文

1. 读 `需求变更日志.md` 最近 3-5 条。
2. 读最新的 `dev/notes/work-log-*.md`。
3. 查看 `问题跟踪.md` 中 Open / In Progress 项。
4. 根据任务需要查看 `系统结构.md` 或其他专项文档。
