# 数据库设计

> 本文档定义禾大 Croda 情报订阅 Agent 的 SQLite 数据模型。设计目标是支持文章采集、正文抽取、结构化打标、证据追踪、客户 watchlist 匹配、活字典维护和月报生成。

---

## 一、设计目标

1. **服务禾大情报场景**：围绕竞品、客户品牌、代理商、成分、技术、功效、法规、市场活动和战略驱动组织数据。
2. **支持多来源采集**：微信公众号、竞品官网、行业媒体、专业数据库、展会和法规源应进入同一文章库。
3. **保留证据链**：每个正式标签必须能追溯到标题、摘要、正文、原文标签、URL 或来源默认信息。
4. **公司自由抽取 + watchlist 匹配**：公司名由 Agent 自由抽取，客户关注名单作为配置和匹配层，不作为 taxonomy。
5. **支持实时活字典**：`ingredient_technology` 允许 `other:<slug>` 内联打标，并按频次驱动晋升正式词表。
6. **支持月报生成**：数据库不仅服务打标，也要能聚合出 TOP10、竞品矩阵、成分趋势、技术创新、活动汇总和附录链接。

## 二、推荐数据流

```text
source_profiles + watchlist_entities
→ article_import_runs
→ articles
→ article_contents
→ article_tag_status
→ v_articles_for_tagging
→ tagging_runs / tagging_attempts
→ article_tags + tag_evidence
→ inline_other_terms + suggested_new_tags + tag_review_queue
→ v_monthly_report_articles / report_runs
→ HTML 月报 + JSON 摘要
```

## 三、核心库表

### 3.1 来源表：`source_profiles`

| 字段 | 类型 | 说明 |
|------|------|------|
| `source_key` | TEXT PRIMARY KEY | 稳定来源 ID |
| `source_name` | TEXT NOT NULL | 来源展示名 |
| `source_type` | TEXT NOT NULL | `wechat_account`、`official_site`、`industry_media`、`database`、`event_site`、`regulator` |
| `source_nature_default` | TEXT | `official_source`、`wechat_public_account`、`media_report_interview`、`third_party_analysis` |
| `url` | TEXT | 官网、公众号主页或数据库入口 |
| `rss_url` | TEXT | RSS 地址，可为空 |
| `ingest_method` | TEXT NOT NULL | `wechat_account`、`direct_rss`、`google_news_rss`、`web_scraper`、`database_query`、`manual_or_reference` |
| `access_status` | TEXT NOT NULL | `working`、`wechat_ok`、`web_scrape`、`blocked_waf`、`blocked_401`、`no_rss` 等 |
| `market_region_json` | TEXT | 来源默认覆盖地区，仅作提示 |
| `language_json` | TEXT | 来源语言 |
| `note` | TEXT | 来源备注 |
| `created_at` / `updated_at` | TEXT | ISO datetime |

### 3.2 客户关注实体表：`watchlist_entities`

客户 Excel 中的行业媒体、竞品、代理商、MNC 品牌、国内品牌进入配置表。它用于匹配和筛选，不限制 Agent 抽取公司。

| 字段 | 类型 | 说明 |
|------|------|------|
| `entity_id` | TEXT PRIMARY KEY | 稳定实体 ID |
| `display_name` | TEXT NOT NULL | 显示名 |
| `aliases_json` | TEXT | 别名、中英文名、品牌名 |
| `entity_role` | TEXT NOT NULL | `self`、`competitor`、`customer`、`channel_partner`、`ecosystem` |
| `market_region_json` | TEXT | `china`、`global`、`europe` 等 |
| `value_chain_stage` | TEXT | 链位置，如 `ingredient_active`、`brand_product` |
| `source_file` | TEXT | 来源 Excel 或人工配置 |
| `status` | TEXT | `active`、`paused`、`archived` |
| `created_at` / `updated_at` | TEXT | ISO datetime |

### 3.3 导入批次表：`article_import_runs`

| 字段 | 类型 | 说明 |
|------|------|------|
| `run_id` | TEXT PRIMARY KEY | 导入批次 ID |
| `input_source` | TEXT | 公众号抓取、官网 scraper、手动导入等 |
| `input_file` | TEXT | 输入文件路径，可为空 |
| `started_at` / `finished_at` | TEXT | 起止时间 |
| `status` | TEXT | `running`、`success`、`failed` |
| `total_count` / `inserted_count` / `updated_count` / `duplicate_count` / `error_count` | INTEGER | 批次统计 |
| `error_log` | TEXT | 错误摘要 |

### 3.4 文章基础表：`articles`

| 字段 | 类型 | 说明 |
|------|------|------|
| `article_id` | TEXT PRIMARY KEY | 稳定 ID，建议由 canonical URL 哈希生成 |
| `canonical_url` | TEXT UNIQUE | 规范化 URL |
| `url` | TEXT NOT NULL | 原始或最终 URL |
| `title` | TEXT NOT NULL | 标题 |
| `summary` | TEXT | 摘要 |
| `published_at` | TEXT | 发布时间，可为空 |
| `fetched_at` | TEXT NOT NULL | 抓取时间 |
| `source_key` | TEXT | 关联 `source_profiles` |
| `source_name_snapshot` | TEXT | 入库时来源名快照 |
| `author` | TEXT | 作者或公众号 |
| `raw_json` | TEXT | 原始抓取数据 |
| `link_status` | TEXT | `valid`、`link_dead`、`blocked`、`unknown` |
| `created_at` / `updated_at` | TEXT | ISO datetime |

推荐索引：`UNIQUE(canonical_url)`、`INDEX(source_key, published_at)`、`INDEX(published_at)`。

| 字段 | 类型 | 说明 |
|------|------|------|
| `article_id` | TEXT PRIMARY KEY | 关联文章 |

| `content_length` | INTEGER | 正文长度 |
| `content_hash` | TEXT | 正文哈希 |
| `extraction_status` | TEXT NOT NULL | `success`、`partial`、`failed`、`link_dead`、`blocked`、`not_attempted` |
| `quality_score` | REAL | 正文质量分，规则派生 |
| `wechat_original_url` | TEXT | 微信原文链接 |
| `raw_tags_json` | TEXT | 原文标签 |
| `extracted_at` | TEXT | 抽取时间 |

| 字段 | 类型 | 说明 |

### 3.7 打标批次与尝试：`tagging_runs` / `tagging_attempts`

`tagging_runs` 记录一批任务，`tagging_attempts` 记录每篇文章每次尝试。

关键字段包括 `tag_run_id`、`agent_name`、`model_name`、`dictionary_version`、`prompt_version`、`run_type`、`status`、`metrics_json`、`attempt_number`、`validation_result`、`post_processing_flags_json`。

## 四、推荐视图

### `v_articles_for_tagging`

稳定提供给打标 runner 的文章包：文章基础字段、正文、来源默认信息、抽取状态、打标状态、raw tags。

### `v_watchlist_matches`

根据 `company` 标签和 `watchlist_entities.aliases_json` 生成匹配结果，用于区分竞品、客户、代理商和生态伙伴。

### `v_monthly_report_articles`

按月份、链接状态、标签、公司角色和复核状态聚合，服务月报生成器。

### `v_inline_other_frequency`

统计 `other:<slug>` 在窗口期内的文章数、来源数、最新出现时间和代表证据，服务人工晋升。

## 五、关键约束

- 不保存 AI 自评 `confidence` 或 `confidence_score`。
- `company` 是自由实体，不受 watchlist 限制。
- `entity_role` 表示公司相对禾大的业务关系，不表示产业链位置。
- `ingredient_technology` 允许 `other:<slug>`；其他封闭枚举字段不得发明正式标签。
- 无证据的正式标签不得写入 `article_tags`。
- 技术失败、内容不足、taxonomy 缺口和业务歧义必须区分处理。
