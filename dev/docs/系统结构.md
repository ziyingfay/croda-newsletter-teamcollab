## 模块说明

| 模块 | 位置 | 当前用途 |
|------|------|----------|
| 正式应用区 | `app/` | 后续放禾大数据源配置、入库、打标 runner、月报生成器；当前保持空白和生产导向 |
| 禾大 prototype | `reference/禾大newsletter/` | 客户提供的 Word prompt、Excel 监测清单和 HTML 月报 prototype |
| 微信/RSS 参考系统 | `reference/wewe-rss/` | 作为微信公众号/RSS 服务化、前后端和数据库实现参考；已删除 `.git` 和 `node_modules` |
| 旧食品饮料归档 | `reference/legacy-food-beverage/` | 旧 TradeMediaRSS 代码、食品饮料字典和旧数据流说明，仅供复用思路 |
| 禾大标签字典 | `dev/docs/标签字段字典.md` | 当前活跃标签字段字典，服务禾大美妆个护原料场景 |
| 打标 skill | `newsletter-tagging/` | 结构化打标流程、prompt、schema、validator 和字典引用 |
| 项目文档 | `dev/docs/` | 项目计划、需求变更、问题、测试、系统、数据库、里程碑 |
| 工作日志 | `dev/notes/` | 保存每次工作上下文 |
| 客户交付物 | `outputs/禾大Croda/` | 需求分析报告和禾大版标签字段字典 |

## 当前活跃数据流

```text
客户数据源配置
→ 公众号/官网/数据库/展会/法规源抓取
→ 文章基础信息入库
→ 正文抽取与链接校验
→ v_articles_for_tagging
→ newsletter-tagging 批量语义打标
→ article_tags + tag_evidence + review_queue
→ 趋势聚合与活字典维护
→ 市场监测月报 HTML + JSON 摘要
```

## 核心标签字段

| 类型 | 字段 |
|------|------|
| 基础字段 | `source_key`、`source_name`、`url`、`published_at`、`access_status`、`ingest_method`、`extraction_status`、`content_language`、`market_region`、`source_nature` |
| Agent 判断字段 | `primary_story_type`、`industry_segment`、`product_application`、`ingredient_technology`、`functional_claim`、`strategic_driver`、`value_chain_stage`、`entity_role`、`company` |
| 审核字段 | `suggested_new_tags`、`review_reasons`、`evidence_records`、`tag_audit` |

## 页面 / 路由

当前正式应用尚未实现 Web 页面。目标月报页面为客户 prototype 对应的交互式 HTML：

- 市场快讯 TOP10
- 执行摘要与行动建议
- 竞品动态监测
- 成分趋势分析
- 技术创新追踪
- 市场活动汇总
- 数据可视化
- 附录原文链接

## API / CLI

当前无正式 API。后续建议最小 CLI 契约如下：

| 命令 | 用途 |
|------|------|
| `python3 dev/scripts/import_articles.py --db croda.sqlite --input ...` | 导入文章 |
| `python3 dev/scripts/run_tagging_batch.py --db croda.sqlite --limit 50 --dry-run` | 打标 dry run |
| `python3 dev/scripts/run_tagging_batch.py --db croda.sqlite --limit 50` | 正式打标 |
| `python3 dev/scripts/build_monthly_report.py --db croda.sqlite --month YYYYMM` | 生成月报 HTML 和 JSON |

## 外部依赖方向

| 依赖方向 | 用途 |
|----------|------|
| 微信公众号抓取链路 | 采集客户核心行业公众号文章 |
| 官网 scraper | 采集竞品官网新闻稿 |
| SQLite | 本地文章库、标签、证据、复核队列和月报聚合 |
| LLM / OpenClaw | 单篇文章语义判断和月报洞察生成 |
| HTML/CSS/JS | 交互式月报、图表、筛选和导出 |

## 后续结构方向

- `reference/wewe-rss/` 可作为公众号服务化和前端管理参考，不直接成为正式代码。
- `reference/legacy-food-beverage/TradeMediaRSS/` 中的 RSS 抓取和正文抽取方法可按需迁移，但必须改为禾大数据源。
- 正式实现应优先从小型 SQLite + CLI runner 开始，跑通采集、入库、打标、月报生成闭环，再考虑 Web 管理端。
