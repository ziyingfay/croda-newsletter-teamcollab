| 模块 | 位置 | 当前用途 |

| 月度工作包 | `outputs/<month>/` | MVP 运行产物：raw、clean、tagging、report JSON、HTML |

## 三、MVP 活跃数据流

```text
raw RSS / 抓取记录
→ rss_clean.json
→ 小龙虾 1：tagging.json
→ report_context.json（近 3 个月最终报告摘要）
→ 小龙虾 2：report_content.json
→ HTML 渲染脚本
→ report.html
```

### 月度工件

| 工件 | 用途 |
|------|------|
| `outputs/<month>/raw/rss_raw_*.json` | 原始抓取记录 |
| `outputs/<month>/raw/fetch_log.json` | 抓取过程与失败日志 |
| `outputs/<month>/rss_clean.json` | 完整干净的当月文章 JSON |
| `outputs/<month>/tagging.json` | 小龙虾 1 打标输出 |
| `outputs/<month>/report_context.json` | 过往 3 个月最终报告摘要 |
| `outputs/<month>/report_content.json` | 小龙虾 2 输出的 HTML 配置文件 |
| `outputs/<month>/report.html` | 最终 HTML 月报 |
| `outputs/<month>/run_log.json` | 运行日志 |

## 四、完整版本数据流（保留）

完整版本仍以 `dev/docs/数据库设计.md` 为准：

```text
taxonomy_terms
source_profiles + watchlist_entities
→ article_import_runs
→ articles + article_contents
→ article_tag_status
→ v_articles_for_tagging
→ tagging_runs / tagging_attempts
→ article_tags + tag_evidence
→ inline_other_terms / tag_review_queue
→ report_runs
```

MVP 不实现上述数据库表，但 JSON 工件字段应与这些表保持可导入关系。

## 五、核心字段与标签

### 基础字段（RSS 清洗脚本写入）

`article_id`、`raw_ref`、`title`、`summary`、`content`、`content_length`、`url`、`canonical_url`、`published_at`、`fetched_at`、`source_key`、`source_name`、`source`、`source_list`、`ingest_method`、`content_language`、`market_region`、`extraction_status`、`raw_tags`。

### Agent 判断标签（小龙虾 1 输出）

`primary_story_type`、`product_application`、`ingredient_technology`、`functional_claim`、`value_chain_stage`、`company`。

### 报告配置字段（小龙虾 2 输出）

`summary`、`sections`、`items`、`charts`、`source_links`、`article_id`、`url`。

## 六、页面 / 路由

当前无 Web 应用路由。MVP 的前端交付是单文件或少量静态资源组成的 HTML 月报：

- 市场快讯 TOP10
- 执行摘要与行动建议
- 竞品动态监测
- 成分趋势分析
- 技术创新追踪
- 重点客户监测
- 市场活动汇总
- 数据可视化
- 附录原文链接

## 七、建议 CLI

MVP 优先：

| 命令 | 用途 |
|------|------|
| `python3 app/scripts/clean_rss.py --month 2606` | 生成 `rss_clean.json` |
| `python3 app/scripts/run_tagging_json.py --month 2606` | 调小龙虾 1，生成 `tagging.json` |
| `python3 app/scripts/build_report_context.py --month 2606 --lookback 3` | 生成 `report_context.json` |
| `python3 app/scripts/run_report_agent.py --month 2606` | 调小龙虾 2，生成 `report_content.json` |
| `python3 app/scripts/render_report.py --month 2606` | 渲染 HTML |

完整版本后续：

| 命令 | 用途 |
|------|------|
| `python3 app/scripts/import_month_json.py --month 2606 --db croda.sqlite` | 把 MVP JSON 导入数据库 |
| `python3 app/scripts/run_tagging_batch.py --db croda.sqlite --limit 50` | 数据库版批量打标 |
| `python3 app/scripts/build_monthly_report.py --db croda.sqlite --month 2606` | 数据库版报告生成 |

## 八、后续结构方向

- 当前先实现 `app/scripts/` 下的 MVP 脚本。
- HTML 模板、CSS、JS 建议放在 `app/report_template/`。
- Agent prompt 和 schema 继续放在 `newsletter-tagging/`，避免业务规则散落。
- 数据库设计不删除；后续通过 `import_month_json.py` 从月度 JSON 接入。
