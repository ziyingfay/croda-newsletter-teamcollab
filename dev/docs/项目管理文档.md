## 一、项目定位

本项目面向**禾大 Croda Beauty**，目标是建立一个按月运行的行业情报订阅 Agent。当前优先级不是先做完整数据平台，而是先以接近人工工作方式的 MVP 跑通：

```text
RSS/公众号抓取
→ 干净 RSS JSON
→ 小龙虾 1 打标 JSON
→ 小龙虾 2 报告内容 JSON
→ HTML 渲染脚本
→ 最终月报
```

数据库设计仍保留为完整版本 / 第二版，不删除、不废弃；但它不作为当前 MVP 的实现前置条件。

## 三、业务目标

| 目标 | 说明 | MVP 实现方式 |
|------|------|--------------|
| 月度报告自动化 | 每月生成交互式 HTML 月报 | `outputs/<month>/report_content.json` 渲染成 HTML |
| 竞争情报 | 监测竞品、新锐原料商、代理商动态 | 小龙虾 1 打标签，小龙虾 2 选入竞品章节 |
| 成分与技术趋势 | 追踪多肽、PDRN、递送、AI、生物技术等 | 标签聚合 + 报告章节写作 |
| 客户品牌需求 | 监测 MNC / 国内品牌的成分、功效、质量趋势 | Watchlist + 重点客户监测章节 |
| 可复核与可迭代 | 保留证据、链接、原始记录 | 月度文件夹保存 raw / clean / tagging / report JSON |

## 五、范围边界

### MVP 包含

- 月度文件夹工作流。
- RSS 原始记录整理与 `rss_clean.json`。
- 小龙虾 1 打标 JSON。
- 小龙虾 2 报告内容 JSON。
- HTML 模板、CSS、JavaScript 与渲染脚本调优。
- 基于近 3 个月最终报告的本期内容选择与去重。
- 人工复核入口和运行日志。

### MVP 暂不包含

- SQLite 入库、数据库状态机和长期历史管理。
- 跨月文章级检索和历史重打标。
- 后台管理界面。
- Agent 直接生成 HTML。
- 复杂数据运营能力。

## 六、阶段计划

| 阶段 | 目标 | 主要任务 | 完成标志 |
|------|------|----------|----------|
| P0 MVP 方案冻结 | 明确月度 JSON 工作流是当前主线 | 更新规划、结构、里程碑；保留数据库第二版 | 文档一致 |
| P1 RSS 清洗 | 生成完整干净 `rss_clean.json` | 合并 raw、去重、补字段、写日志 | `outputs/2606/rss_clean.json` 可用 |
| P2 打标 JSON | 小龙虾 1 跑通 | 读取 clean JSON、输出 tagging JSON、校验 | `tagging.json` 通过 validator |
| P3 HTML 渲染 | 固定报告版式 | 定义 `report_content.json`、拆模板、调 CSS/JS | 改 JSON 可稳定输出 HTML |
| P4 报告 Agent | 小龙虾 2 跑通 | 读近 3 月报告 + 当月 JSON，生成报告配置 | `report_content.json` 可渲染 |
| P5 客户验收 | 确认分类、标签、内容标准、HTML | 交付 2606 样例，按反馈调整 | 客户确认第一版 |
| P6 完整版本评估 | 判断是否接入数据库 | 评估月度 JSON 是否够用 | 决定是否进入 SQLite 实现 |

## 七、验收标准

- 月度文件夹中每个工件路径固定、可读、可复核。
- `rss_clean.json` 中文章字段完整，失败记录不混入文章列表。
- `tagging.json` 与 `newsletter-tagging` schema 一致。
- `report_content.json` 可被渲染脚本稳定输出 HTML。
- HTML 支持客户确认的栏目、折叠、筛选、图表、链接和打印/PDF 友好。
- 每条报告内容可追溯到 `article_id` 和原文 URL。
- MVP JSON 字段可通过 adapter 导入未来数据库设计。
