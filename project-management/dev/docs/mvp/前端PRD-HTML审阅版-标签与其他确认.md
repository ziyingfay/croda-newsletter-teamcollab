# 前端 PRD · HTML 月报审阅版（标签展示 / 其他-X 待确认 / 人工补充标签）

> 给前端改造用。背景：后端已定版——**栏目由 AI 判断（平等多选），标签由脚本判断，后处理补"其他-X 待确认"**。
> 本 PRD 只描述**前端 HTML 审阅版**要怎么改，配合后端这套机制。依据：
> 字典 V4 §一/§九、`脚本层-清洗入库与脚本打标-开发需求与设计.md` §3.1b/§3.3b。
>
> 一句话目标：让客户在审阅版里**看得懂标签、能筛出"待确认"内容、能手动补标签**，且这些人工动作
> **沉淀成修改记录**、不污染正式字典。

---

## 0. 前端会拿到的数据（后端契约）

每篇文章对象里与前端相关的字段：

| 字段 | 来源 | 说明 |
|---|---|---|
| `section.sections[]` | AI | 平等多选栏目（competitor_watch / ingredient_innovation=新成分与新技术 / ka_watch / market_event / regulation_policy）。一篇可属多个 |
| `relevance` | AI | relevant / not_relevant / unclear |
| 脚本标签（company / ingredient / functional_claim / product_application / is_event…） | 脚本 | 展示/筛选用 |
| **`其他-成分技术` / `其他-宣称` / `其他-活动` / `其他-法规`** | 后处理 | **待确认入口**：AI 归入该栏目但脚本没命中具体标签（`其他-成分技术`、`其他-宣称` 仅挂"新成分与新技术"栏目下） |
| `report_guidance` | AI（可选） | 偶尔的一句提醒（如"仅活动预告，建议一句话带过"） |

> 注意：`其他-X` 是**通用占位**，不是具体成分名；前端不要把它当成"系统识别出的成分"。

---

## 1. 「其他-X」不要平铺在正文卡片上

**问题**（来自讨论）：正文卡片上到处显示一堆 `Other` / `其他`，客户看了不知道是什么，像系统出错。

**要求**：
1. **正文卡片默认不显示**裸 `Other` / `其他` 文本标签。
2. 如果要在卡片上体现，**必须显示成明确的"待审核"措辞**，例如 chip 文案：
   `🔍 其他：成分/技术`、`🔍 其他：宣称`、`🔍 其他：活动`、`🔍 其他：法规`——让客户知道"这是需要人工看的内容",
   而不是系统错误或一个看不懂的泛标签。
3. 颜色/样式与正式标签区分（如灰色/虚线边框 + 放大镜图标）。

---

## 2. 修改「标签筛选区」
新增
- **「待确认」筛选组**：
  - `其他-成分技术`（待确认）
  - `其他-宣称`（待确认）
  - `其他-活动`（待确认）
  - `其他-法规`（待确认）
  让客户**一键筛出所有需要人工判断的文章**，集中处理。

> 价值：客户审阅时先点"其他-成分技术"，就能把"可能含字典外新成分"的文章挑出来逐条看，
> 而不是在几百张卡片里大海捞针。

---

## 3. 人工补充标签（审阅版可手动加，但不进正式字典）

每张卡片提供「**+ 补充标签**」入口，允许客户在审阅时手动加标签（如客户认为该文应标"玻色因"）。规则：

1. 人工补充标签**单独存放**（如 `manual_tags[]` 或修改记录），与脚本/AI 标签**分开**，前端样式也要区分（如"人工"角标）。
2. **不自动进入正式字典**、**不让 AI 维护**。
3. 持久化到**审阅/修改记录**（见 §4），不回写后端的标签配置文件。
4. 对 `其他-X 待确认`，客户的处理动作应支持：**保留 / 删除 / 手动注明具体是哪个成分技术 / 标记"建议入字典"**。

---

## 4. 人工修改记录（沉淀，供后续迭代字典）

- 所有人工动作（补标签、删标签、把"其他-X"注明为具体成分、勾选"建议入字典"）写入**本期审阅修改记录**
  （随月报 HTML 版本保存，如 `report/<YYYYMM>/review-edits.json`）。
- **当前不做活字典**：不在本期把人工补的标签合进正式字典。
- 跑几期后，由人统一看"客户反复补了哪些标签 / 反复把哪些'其他'注明成同一个成分"，**再决定是否纳入正式字典**。
- 这条是"当前不做活字典，先沉淀人工修改记录，后续再迭代"的落地。

---

## 6. 验收要点（前端自测）

1. 卡片上无裸 `Other`；待确认内容显示为"🔍 其他：X"且样式可辨。
2. 筛选区能单独筛出 `其他-成分技术 / 其他-活动 / 其他-法规`及 `其他-宣称`。
4. 人工补充/修改的标签与系统标签视觉区分，且写入审阅修改记录文件。
5. 人工动作不触发任何对后端标签配置（watchlist/ingredient-alias 等）的写入。
