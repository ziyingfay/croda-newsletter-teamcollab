# 标签字段字典

> 本文档定义食品饮料 newsletter 文章入库、基础识别、Agent 语义打标、客户匹配和人工复核时使用的字段体系。当前版本采用“两层模型”：基础数据字段由代码或轻量规则优先处理；深度标签由 Agent 阅读文章后判断。公司名是自由实体抽取结果，不进入字典词表。

---

## 设计原则

1. **两层处理**：先由代码处理基础字段，再由 Agent 判断需要理解文章本意的标签。
2. **MECE 优先**：每个字段代表一个独立维度；相同含义不得跨字段重复表达。
3. **客户匹配导向**：字段必须帮助后端把文章精准匹配给客户关注的行业、市场、价值链、战略议题或公司。
4. **不盲目细分**：如果一个字段不能稳定提升判断或筛选价值，不进入字典。
5. **公司不入字典**：公司名由 Agent 从文章中自由提取，只写入打标 JSON 和数据库，不作为永久 taxonomy 标签。
6. **证据驱动**：Agent 判断字段必须有标题、摘要、正文、原文标签或 URL 证据。
7. **无 confidence 口径**：不使用 AI 自评 confidence；质量控制依靠 schema validation、证据校验、post-processing 和 review queue。

## 一、两层字段结构

### 1.1 基础数据字段标签

这类字段不需要 Agent 通读全文，优先由 Python 脚本、来源配置、URL、元数据、短文本扫描或正文抽取状态判断。Agent 可以在证据明显冲突时请求复核，但不应把大量 token 用在这些字段上。

| 字段 | 类型 | 处理方式 | 用途 |
|------|------|----------|------|
| `source_key` | string | 来源配置 / 入库脚本 | 稳定来源 ID |
| `source_name` | string | 来源配置 / RSS | 来源展示名 |
| `url` / `canonical_url` | string | 入库脚本 | 去重、追溯和抓取 |
| `published_at` | datetime/null | RSS / 解析脚本 | 时间筛选 |
| `fetched_at` | datetime | 抓取脚本 | 抓取批次追踪 |
| `access_status` | enum | 来源配置 | RSS 可用性和抓取状态 |
| `ingest_method` | enum | 来源配置 / URL | 直接 RSS、Google News RSS、评论 feed 等 |
| `extraction_status` | enum | 正文抽取脚本 | 正文质量与是否需要复核 |
| `content_language` | enum | 语言检测 / 轻量规则 | 语言筛选 |
| `market_region` | tag[] | 来源配置 / 地名扫描 / URL | 地区筛选；仅在显式证据存在时写入 |
| `source_nature` | enum | 来源配置 / URL / 署名规则 | 信息来源性质 |
| `raw_tags` | string[] | 原文标签 / RSS | 原始标签保存，不直接等于正式标签 |

### 1.2 Agent 判断标签

这类字段需要 Agent 理解文章主旨后判断。每个字段服务不同维度，不应互相替代。

| 字段 | 类型 | 用途 | 约束 |
|------|------|------|------|
| `primary_story_type` | enum | 判断“这篇文章主要是什么类型的事件/内容” | 单选，必须有一个主类型 |
| `industry_segment` | enum | 判断文章主要属于食品饮料产业的哪个大段 | 单选；跨多个大段时用 `cross_sector` |
| `product_application` | tag[] | 判断文章涉及的产品或应用场景 | 多选，只有明确产品/应用时填写 |
| `strategic_driver` | tag[] | 判断文章对客户为什么重要 | 多选，建议 1-3 个 |
| `value_chain_stage` | enum | 判断文章主要影响产业链哪个位置 | 单选；跨链条时用 `cross_chain` |
| `company` | string[] | 提取文章事件涉及的公司实体 | 自由文本，不受字典词表限制 |
| `suggested_new_tags` | object[] | 现有 taxonomy 不足时提出新标签建议 | 进入复核队列，不直接入正式标签 |

## 三、基础数据字段词表

### 3.1 来源状态 `access_status`

| 标签值 | 中文名 | 定义 |
|--------|--------|------|
| `working` | 正常 RSS | 可直接通过 RSS 获取内容 |
| `google_news` | Google News RSS | 官网 RSS 不可用，但可通过 Google News RSS 获取 |
| `blocked_waf` | WAF 封锁 | 403、CloudFlare、企业级 WAF 等阻断 |
| `blocked_401` | 认证阻断 | 需要登录、订阅或认证 |
| `no_rss` | 无 RSS | 未找到可用 RSS，Google News 也无结果 |
| `blocked` | 泛化封锁 | 历史旧值，不建议新增使用 |
| `other` | 其他 | 出现新的抓取状态，需人工确认 |

### 3.2 抓取方式 `ingest_method`

| 标签值 | 中文名 | 定义 |
|--------|--------|------|
| `direct_rss` | 直接 RSS | 使用媒体官网或官方托管 RSS |
| `google_news_rss` | Google News RSS | 使用 Google News 搜索 RSS 作为替代源 |
| `comment_feed` | 评论 Feed | 仅能抓到评论 RSS，不代表主内容 |
| `manual_or_reference` | 手动/参考补充 | 来自人工整理、公开转载页、旧样本或参考材料 |
| `unavailable` | 暂不可抓取 | 无 RSS、认证、WAF 或网站不可访问 |
| `other` | 其他 | 抓取方式无法归入现有类别 |

### 3.3 抽取状态 `extraction_status`

| 标签值 | 中文名 | 定义 |
|--------|--------|------|
| `success` | 成功 | 正文可用 |
| `partial` | 部分成功 | 有正文但可能含导航、广告或缺失明显 |
| `failed` | 失败 | 未能获取正文 |
| `blocked` | 被阻断 | WAF、登录、验证码、401/403 等 |
| `skipped` | 跳过 | 已存在、重复或不需抽取 |
| `not_attempted` | 未尝试 | 尚未进入正文抽取流程 |
| `other` | 其他 | 新处理状态，需人工确认 |

### 3.4 语言 `content_language`

| 标签值 | 中文名 | 定义 |
|--------|--------|------|
| `en` | 英语 | 正文主体为英语 |
| `zh` | 中文 | 正文主体为中文 |
| `de` | 德语 | 正文主体为德语 |
| `fr` | 法语 | 正文主体为法语 |
| `es` | 西班牙语 | 正文主体为西班牙语 |
| `it` | 意大利语 | 正文主体为意大利语 |
| `id` | 印尼语 | 正文主体为印尼语 |
| `th` | 泰语 | 正文主体为泰语 |
| `fil` | 菲律宾语 | 正文主体为菲律宾语 |
| `mixed` | 混合语种 | 多语正文或稳定双语输出 |
| `unknown` | 未知 | 内容不足以判断语言 |
| `other` | 其他 | 已确认语言但当前词表未列入 |

### 3.5 地区 `market_region`

`market_region` 只在文章或来源明确指向地区时填写。若只是来源媒体所在地，不等于文章讨论市场。

| 标签值 | 中文名 | 定义 |
|--------|--------|------|
| `global` | 全球 | 明确讨论全球市场、多地区趋势或无单一区域限制 |
| `europe` | 欧洲 | 欧洲整体、欧盟或欧洲国家群 |
| `north_america` | 北美 | 美国、加拿大或北美市场 |
| `sea` | 东南亚 | 东南亚国家或跨多个东南亚市场 |
| `nee` | 东北亚 | 除中国外的东北亚/北亚相关市场，如日本、韩国等 |
| `middle_east` | 中东 | 中东或海湾地区 |
| `china` | 中国 | 中国大陆及以中国市场为核心的内容 |
| `australia` | 澳洲 | 澳大利亚市场 |
| `indonesia` | 印尼 | 印尼市场 |
| `thailand` | 泰国 | 泰国市场 |
| `philippines` | 菲律宾 | 菲律宾市场 |
| `italy` | 意大利 | 意大利市场 |
| `germany` | 德国 | 德国市场 |
| `uk` | 英国 | 英国市场 |
| `other` | 其他 | 明确地区存在但未列入词表 |

### 3.6 来源性质 `source_nature`

| 标签值 | 中文名 | 定义 | 排除条件 |
|--------|--------|------|----------|
| `official_source` | 官方消息来源 | 公司官网/新闻稿、监管机构、协会、展会主办方、品牌官方平台等一手发布 | 媒体转述官方信息且有编辑加工时不用 |
| `media_report_interview` | 媒体报道/采访 | 行业媒体、新闻媒体、编辑报道、人物采访、专家采访、现场报道 | 纯咨询报告或市场研究不用 |
| `wechat_public_account` | 微信公众号 | 微信公众号原文、公众号转载或公开转载页 | 普通中文网站不因语言为中文而归入此类 |
| `third_party_analysis` | 第三方分析 | 咨询公司、市场研究机构、数据平台、智库或第三方分析文章 | 新闻只引用报告一句话时仍按报道判断 |
| `other` | 其他 | 无法可靠判断或出现新的稳定来源类型 | 需进入复核 |

## 四、Agent 判断标签词表

### 4.1 主故事类型 `primary_story_type`

`primary_story_type` 回答“这篇文章主要在报道什么类型的事情”。它是单选字段，用于避免同一篇文章被 business、innovation、trend 等重复含义标签淹没。

| 标签值 | 中文名 | 定义 | 不应使用的情况 |
|--------|--------|------|----------------|
| `business_transaction` | 商业动作 | 并购、融资、投资、合作、产能、财报、组织调整、市场进入 | 只是产品介绍不用 |
| `product_launch_or_update` | 产品/配方发布 | 新品、配方、包装、口味、应用方案、商业化发布或升级 | 单纯趋势预测不用 |
| `regulation_policy` | 政策法规 | 法规、标准、认证、标签、政策、监管动作 | 企业普通合规声明不用 |
| `research_science` | 科研科学 | 学术研究、临床、实验、科学证据、技术验证 | 市场调查不用 |
| `market_consumer_insight` | 市场/消费者洞察 | 市场规模、品类走势、消费者偏好、趋势报告、消费数据 | 单一公司新闻不用，除非核心是市场洞察 |
| `technology_process_innovation` | 技术/工艺创新 | 新工艺、设备、数字化、AI、加工、包装或配料技术突破 | 只是提到技术名不用 |
| `safety_quality_issue` | 安全/质量事件 | 召回、污染、风险评估、限量、质量争议、检测问题 | 普通健康宣称不用 |
| `supply_chain_trade` | 供应链/贸易 | 原料供应、价格、关税、进出口、物流、地缘与气候供应风险 | 国内渠道扩张不用 |
| `event_news` | 活动/会议 | 展会、峰会、论坛、研讨会预告或会后报道 | 活动只是背景时不用 |
| `other` | 其他 | 文章相关但无法归入上述故事类型 | 进入复核或 suggested_new_tags |

### 4.2 产业大段 `industry_segment`

`industry_segment` 回答“文章主要服务食品饮料产业的哪个大段”。它是单选字段，跨多个大段且无法确认主线时用 `cross_sector`。

| 标签值 | 中文名 | 定义 |
|--------|--------|------|
| `finished_food` | 成品食品 | 食品品牌、成品食品、零食、烘焙、乳制品、预制食品等 |
| `beverage` | 饮料 | 酒精/非酒精饮品、茶咖、功能饮料、瓶装水、RTD 等 |
| `ingredients` | 配料/原料 | 食品配料、添加剂、风味、色素、蛋白、甜味剂、功能成分 |
| `nutrition_supplements` | 营养保健 | 膳食补充剂、功能营养、健康食品、营养品市场 |
| `packaging_processing` | 包装与加工 | 包装材料、设备、加工工艺、保鲜、制造技术 |
| `agriculture_commodities` | 农业与大宗原料 | 农作物、畜牧、水产、可可、咖啡、糖、谷物等上游供应 |
| `foodservice_retail` | 餐饮零售 | 餐饮、商超、便利店、电商、即时零售和渠道 |
| `petfood_feed` | 宠物食品/饲料 | 宠物食品、宠物营养、动物饲料 |
| `cross_sector` | 跨产业 | 同时覆盖多个大段且没有明显主线 |
| `other` | 其他 | 食品饮料相关但无法归入现有大段 |

### 4.3 产品/应用场景 `product_application`

`product_application` 只在文章明确涉及产品、品类或应用场景时填写。它不是必填字段，不用于泛化地重复 `industry_segment`。

| 标签值 | 中文名 | 定义 |
|--------|--------|------|
| `beverage` | 饮料应用 | 饮料产品或饮料配方应用 |
| `dairy` | 乳制品 | 牛奶、酸奶、奶酪、乳饮、乳清应用 |
| `bakery_snacks` | 烘焙/零食 | 面包、糕点、饼干、零食等 |
| `confectionery` | 糖果巧克力 | 糖果、巧克力、甜食、口香糖 |
| `meat_seafood` | 肉类/水产/蛋 | 肉、禽、蛋、水产及其替代或加工 |
| `plant_based_alt_protein` | 植物基/替代蛋白 | 植物肉、植物奶、培养食品、替代蛋白应用 |
| `ready_meals` | 预制/方便食品 | 预制菜、即食餐、冷冻餐、方便食品 |
| `sauces_condiments` | 酱料/调味品 | 汤料、酱料、调味品、蘸酱 |
| `infant_child_nutrition` | 婴童营养 | 婴幼儿配方、儿童营养食品 |
| `pet_food` | 宠物食品 | 宠物主粮、零食、宠物营养 |
| `ingredients_general` | 通用配料应用 | 文章谈配料能力但没有明确下游品类 |
| `packaging_materials` | 包装材料应用 | 包装材料、包装容器、可持续包装应用 |
| `other` | 其他 | 明确产品应用但未列入 |

### 4.4 战略驱动 `strategic_driver`

`strategic_driver` 回答“这篇文章为什么对客户有业务意义”。它与 `primary_story_type` 独立：例如一篇新品发布的战略驱动可以是健康营养，也可以是可持续、风味或成本。

| 标签值 | 中文名 | 定义 | 排除条件 |
|--------|--------|------|----------|
| `health_nutrition` | 健康营养 | 营养、功能、补充剂、GLP-1、蛋白、肠道、免疫、健康老龄化等 | 只出现“健康”营销词但无具体健康逻辑时不用 |
| `sustainability` | 可持续 | 减碳、循环、再生农业、废弃物利用、可持续供应、环保包装 | 只有 vague green/eco/natural 不用 |
| `regulation_compliance` | 法规合规 | 合规压力、政策适配、标签、认证、监管风险 | 文章主类型已是法规时，只有客户匹配需要时才重复使用 |
| `cost_margin_value` | 成本/利润/价值 | 降本、价格压力、利润率、通胀、性价比、采购成本 | 单纯销售增长不用 |
| `premium_indulgence` | 高端/享受 | 高端化、纵享、质感、风味体验、情绪价值 | 普通新品不用 |
| `convenience` | 便利/效率 | 即食、即饮、省时、渠道便利、操作效率 | 普通包装变化不用 |
| `sensory_flavor` | 感官/风味 | 味道、香气、口感、质构、风味趋势、感官体验 | 普通产品名提到口味不用 |
| `clean_label_natural` | 清洁标签/天然 | 短配料表、天然来源、少添加、无人工添加 | 可持续主题不要自动归入此项 |
| `supply_resilience` | 供应韧性 | 供应稳定、替代原料、气候风险、地缘风险、原料短缺 | 普通供应商新闻不用 |
| `digital_ai` | 数字化/AI | AI、自动化、数据平台、数字供应链，且是文章核心 | AI 只是路人式提及不用 |
| `market_expansion` | 市场扩张 | 新地区、新渠道、新客群、新品类增长机会 | 普通公司增长不用 |
| `other` | 其他 | 有明确业务驱动但未列入 |

### 4.5 价值链阶段 `value_chain_stage`

`value_chain_stage` 回答“文章主要影响产业链哪个位置”。它是单选字段。

| 标签值 | 中文名 | 定义 |
|--------|--------|------|
| `agriculture_sourcing` | 农业/采购 | 农业生产、原料采购、上游种养殖、大宗商品 |
| `ingredients_formulation` | 配料/配方 | 配料开发、配方应用、风味、功能成分、技术应用 |
| `manufacturing_processing` | 制造/加工 | 工厂、设备、加工工艺、产能、食品制造 |
| `packaging` | 包装 | 包装材料、包装技术、包装法规、包装线 |
| `distribution_retail` | 分销/零售 | 物流、渠道、商超、电商、便利店、即时零售 |
| `foodservice` | 餐饮 | 餐厅、咖啡茶饮、菜单、餐饮供应链 |
| `consumer_market` | 消费者/市场 | 消费者行为、市场数据、人群洞察、品类趋势 |
| `regulation_research` | 监管/科研 | 政府监管、标准制定、科研机构、实验验证 |
| `cross_chain` | 跨链条 | 明确覆盖多个阶段且无主阶段 |
| `other` | 其他 | 价值链位置明确但未列入 |

### 4.6 公司 `company`

`company` 是文章中由 Agent 抽取的公司实体名称，不是字典标签。它用于回答“某家公司最近在做什么”，也用于客户指定竞品、客户、供应商或目标公司匹配。

#### 写入规则

只有在公司是新闻事件主体或重要相关方时才写入：

- 标题、摘要或正文核心段落直接讨论该公司。
- 公司发生了新品、并购、投资、合作、召回、财报、可持续、供应链、监管、研发或市场动作。
- 文章围绕该公司的高管、品牌、子公司或明确归属业务展开。
- 第三方分析中该公司是被重点分析的案例或样本。

以下情况不写入：

- 只是引用该公司某位员工一句话，但事件本身与该公司无关。
- 只在广告、导航、相关阅读、赞助列表、长公司名单中出现。
- 公司名只是行业背景或历史例子，不是当前新闻重点。
- 品牌与母公司关系不明确，且上下文不能确认公司实体。

#### 存储建议

Agent 输出应保留公司显示名，如 `Coca-Cola`、`Nestlé`、`Givaudan`。入库脚本可另外生成标准化键：

```json
{
  "company": ["Coca-Cola", "Givaudan"],
  "company_normalized": ["coca_cola", "givaudan"]
}
```

公司名不进入 `suggested_new_tags`，不需要每周 qualification，也不回写本文档词表。后端可直接基于 JSON / 数据库中的公司字段进行查询、聚合和客户匹配。

## 五、Suggested New Tags 与字典维护

当文章相关，但现有 Agent 判断标签无法准确表达其长期筛选价值时，Agent 可以返回 `suggested_new_tags`。这只适用于 taxonomy 标签，不适用于公司名。

### 5.1 使用条件

1. 文章内容明确属于某个现有字段的判断范围。
2. 正式词表中没有合适标签。
3. 新标签对 newsletter 筛选、栏目归类或客户匹配有长期价值。
4. 能提供标题、摘要、正文、URL 或原文标签证据。

如果只是证据不足、正文缺失、语义模糊，不要建议新标签，应进入 `needs_review`。

### 5.2 候选结构

```json
{
  "field": "strategic_driver",
  "label": "tariff_shock",
  "display_name": "关税冲击",
  "parent_label": "supply_resilience",
  "reason": "多篇文章讨论关税对食品供应链、进出口和成本的影响，现有标签只能泛化表达。",
  "evidence": "title/summary/content",
  "status": "pending_review"
}
```

### 5.3 人工审核规则

候选标签必须满足至少两条，才建议正式入库：

- 一周内出现不少于 2 篇不同文章，或来自不少于 2 个不同来源。
- 不能被现有标签准确表达。
- 能显著提升 newsletter 筛选、栏目归类或客户匹配。
- 标签定义边界清晰，Agent 可以稳定复现。
- 不是一次性公司名、活动名、产品名或过窄事件。

## 六、AI 打标约束

### 6.1 推荐判断顺序

1. 读取基础字段，不重新推理已由代码确定的字段，除非发现明显冲突。
2. 判断文章是否与食品饮料行业 intelligence 相关。
3. 判断 `source_nature` 是否可信。
4. 判断单选 `primary_story_type`。
5. 判断单选 `industry_segment`。
6. 仅在有明确证据时填写 `product_application`。
7. 判断 1-3 个 `strategic_driver`。
8. 判断单选 `value_chain_stage`。
9. 抽取文章事件主体或重要相关方公司到 `company`。
10. 为每个 Agent 判断字段提供 evidence record。
11. 若 taxonomy 缺失长期有价值标签，写入 `suggested_new_tags`；不要创造正式标签。

### 6.2 防重复规则

- 不把 `primary_story_type=market_consumer_insight` 和 `strategic_driver=market_expansion` 同时用于同一层意思；前者是文章类型，后者是战略原因。
- 不把 `industry_segment=packaging_processing` 和 `value_chain_stage=packaging` 当作同一判断。前者表示文章所属产业大段，后者表示影响链条阶段。若两者相同是因为文章确实属于包装产业且影响包装阶段，允许共存。
- 不把 `sustainability` 同时当作故事类型和战略驱动；它只能是 `strategic_driver`。
- 不把公司名、品牌名、展会名放入 taxonomy 标签；公司名只进入 `company`。
- 不为了满足标签数量而补标签；宁可少打，也不要弱证据打标。

### 6.3 证据规则

每个 Agent 判断字段都必须有证据。证据优先级：

1. `content`
2. `title`
3. `summary`
4. `raw_tags`
5. `url`
6. `source_defaults`

`source_defaults` 只能辅助基础字段，不能单独支撑深度语义标签。
