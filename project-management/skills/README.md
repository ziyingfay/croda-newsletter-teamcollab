# Internal Project-Management Skills

This directory is for internal project-management, repository-organization, collaboration, and governance skills.

Customer-deliverable Xiaolongxia/OpenClaw skills that the product workflow installs or reads belong under `../../product/skills/`, not here.

| Directory | Purpose |
|-----------|---------|
| `active/` | Current internal project-management skills |
| `drafts/` | Skill drafts |
| `examples/` | Skill usage examples |
| `test-fixtures/` | Test inputs and fixtures |
| `archive/` | Older or retired versions |

Included active skill:

- `active/croda-project-management/`: Croda / Heda project directory rules, Python backend guardrails, RSS JSON dual naming, public output handoff rules, and monthly newsletter workflow.
